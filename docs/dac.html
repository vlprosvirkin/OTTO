Architecture Spec

Decentralized Autonomous Corporation

How autonomous treasury management works — from fund creation to invoicing, payroll, budgeting, yield optimization on idle balances, and emergency controls

Participants

💰

Investor / LP Holder

Deposits funds into the treasury, receives LP tokens representing their share, and governs the corporation through token-weighted voting. A single role — every contributor is also a decision-maker.

📊

Manager

Operates the treasury — issues invoices, approves expenses, runs payroll, and manages budgets. Cannot withdraw funds directly.

🛡

Guardian

Emergency oversight. Can freeze the entire corporation in case of an emergency. Controls global safety parameters.

Corporation Lifecycle

1

Corporation Creation (Factory)

The Factory deploys a complete set of contracts for each DAC:

  • Pool (Treasury) — holds all corporation assets
  • LP Token — non-transferable share of participation
  • Voting — governance and decision-making module
  • Configuration — corporation rules (fees, limits, windows, budgets)

After creation, the Factory renounces ownership — the corporation becomes fully self-governing.

2

Fundraising

Investor → deposits funds → Pool values contribution via oracle → → mints LP tokens proportional to deposit → minus entry fee to Manager

Constraints:

  • Only during the designated fundraising window
  • Minimum and maximum deposit size enforced
  • Private corporations — whitelist-only addresses
  • LP tokens are frozen for N hours after deposit
3

Treasury Operations (Manager)

Manager → submits an expense → Pool validates: ✓ Recipient is on the approved payee whitelist ✓ Expense category is within the approved budget ✓ Amount is within per-tx and daily spending caps ✓ Invoice reference is attached and valid → Pool executes the payment on behalf of the corporation

The Manager cannot withdraw assets freely — every outflow must pass through approved channels: invoices, payroll schedules, or budgeted expense categories. Validators decode each transaction and verify its legitimacy.

Operation types:

  • Invoices — pay external vendors and service providers against submitted invoices
  • Payroll — recurring salary payments to employees on a set schedule
  • Budgeting — allocate funds to departments or categories with spending caps
  • Expense approvals — one-off payments within pre-approved budget lines
4

Yield Optimization (Idle Balance Management)

Pool monitors idle USDC balance → ✓ Calculates reserve needed for upcoming payroll + invoices ✓ Sweeps excess USDC into USYC (tokenized US T-bills) ✓ USYC accrues yield continuously (~5% APY) ✓ Auto-redeems USYC back to USDC when liquidity is needed

Idle treasury balances are never left sitting unproductive. The Pool automatically invests surplus USDC into USYC (Hashnote tokenized T-bills), earning risk-free yield on idle funds while maintaining enough liquid USDC to cover short-term obligations.

How it works:

  • Reserve threshold — Pool keeps enough liquid USDC to cover N days of projected expenses
  • Auto-sweep — any balance above the threshold is deposited into USYC
  • Auto-redeem — when liquid balance drops below threshold, USYC is redeemed back to USDC
  • Yield accrual — USYC appreciates in value continuously, increasing the total treasury value
Example: $100k treasury, $20k monthly burn → Reserve: $30k USDC (liquid, 1.5x monthly burn) → Invested: $70k in USYC earning ~5% APY → Yield: ~$3,500/year on idle funds
5

Withdrawal

Investor → requests withdrawal → Pool calculates: 1. Proportional share of all assets 2. Performance fee (if LP price increased since entry) 3. Rage quit fee (if outside withdrawal window) → Burns LP tokens → Transfers asset share to investor

6

Manager Fees

Fee When Logic
Entry fee On deposit % of minted LP tokens
Management fee Continuous Annual % of AUM, accrued every second
Performance fee On investor withdrawal % of profit (if LP price increased)

The Manager claims fees via a separate transaction — receiving a proportional share of all pool assets.

7

Governance (Voting)

LP Holder → creates proposal → Token-weighted vote → → If quorum reached and majority in favor → Pool executes action

What can be proposed:

  • Replace the Manager (if allowed by configuration)
  • Remove a payee from the approved whitelist
  • Adjust budget caps or spending limits
  • Add or remove approved expense categories
  • Change yield reserve threshold or disable auto-sweep

Constraints:

  • The Manager can submit any type of proposal
  • Regular investors — only the actions listed above
  • A minimum LP token share is required to create a proposal
  • Votes are counted by snapshot (at the moment of proposal creation)
8

Emergency Stop

Guardian → activates emergency stop → ✗ All operations blocked (invoices, payroll, expenses) ✗ Deposits blocked ✗ New proposals blocked ✓ Withdrawals WITHOUT penalty — allowed

Core Principle

The Manager operates, but cannot extract funds. Idle treasury balances are never left unproductive — surplus USDC is automatically swept into USYC (tokenized T-bills) to earn yield. Investors cannot run day-to-day operations, but can vote to replace the Manager or restrict their authority. The Guardian can freeze everything in an emergency.
